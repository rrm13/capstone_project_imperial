# Datasheet for Dataset: BBO Optimization History

## Motivation

### Purpose
This dataset captures the historical interaction history of a Black-Box Optimization (BBO) challenge. It records query vectors submitted across multiple rounds and the corresponding objective outputs returned by the evaluation system.

### Intended Use
- Bayesian Optimization research
- Surrogate model training
- Acquisition function evaluation
- Optimization benchmarking
- Convergence analysis

## Dataset Composition

### Dataset Contents
The dataset consists of:

1. Input files (`inputs_X.txt`)
   - Query vectors submitted at each optimization round
   - Multiple independent tasks per round
   - Variable-dimensional parameter vectors

2. Output files (`outputs_X.txt`)
   - Objective function values
   - One score per task
   - Generated by the challenge evaluation environment

### Unit of Observation
A single observation consists of:

- Round number
- Task identifier
- Input vector
- Objective value

### Number of Tasks
The challenge contains 8 optimization tasks.

### Feature Dimensions
Observed dimensions include:

- 2D
- 3D
- 4D
- 5D
- 6D
- 8D

### Value Ranges
Input variables are assumed to be normalized to:

[0,1]

Output values may be:

- Positive
- Negative
- Near-zero
- Large magnitude values

## Data Collection Process

### Collection Method
Data is collected iteratively:

1. Candidate query vectors are generated.
2. Queries are submitted to the BBO environment.
3. Objective values are returned.
4. Results are appended to the optimization history.

### Sampling Strategy
Sequential adaptive sampling using Bayesian Optimization.

### Time Horizon
The dataset grows incrementally with every optimization round.

## Preprocessing

### Input Processing
- Vectors stored as NumPy-style arrays.
- Values generally normalized to [0,1].
- Parsed into numerical arrays before model fitting.

### Output Processing
- Objective values extracted from NumPy float representations.
- Used directly as optimization targets.

## Data Quality

### Strengths
- Real optimization trajectories.
- Multi-round observations.
- Multiple task dimensions.
- Suitable for convergence analysis.

### Limitations
- Small sample size per task.
- Sequential sampling bias.
- Potential objective noise.
- Not independently and identically distributed (IID).

## Recommended Uses

- Bayesian Optimization evaluation
- Gaussian Process research
- Expected Improvement studies
- Optimization visualizations
- Surrogate model benchmarking

## Not Recommended Uses

- Classification tasks
- Population statistics
- Causal inference
- Fairness analysis
- Safety-critical decision systems

## Maintenance

### Update Frequency
Updated after each optimization round.

### Versioning
A new version can be produced after every completed round.

### Retention
Historical rounds should be preserved to support reproducibility and convergence analysis.

## Ethics and Privacy

The dataset:

- Contains no personal data.
- Contains no sensitive attributes.
- Contains only optimization inputs and outputs.
- Presents minimal privacy risk.
